# 3. JavaScript Object কেন প্রয়োজন

একটি application-এর data দরকার।

আগের lessonগুলোতে data সাধারণত simple variable-এর মাধ্যমে রাখা হয়েছে:

```js
const age = 40;
const name = "Tapas";
const isAdmin = true;
```

এগুলো `number`, `string`, `boolean`-এর মতো **primitive data type**।

কিন্তু real-world application-এ data সাধারণত আরও complex হয়। যেমন:

- User data
- Employee data
- Company data
- Product data
- Student data
- Address data

একজন user-এর শুধু একটি `name` থাকে না। তার থাকতে পারে:

- name
- age
- email
- address
- role
- preferences
- methods বা behavior

এগুলো আলাদা আলাদা variable-এ রাখলে data ছড়িয়ে যায় এবং manage করা কঠিন হয়।

এই ধরনের related এবং structured data একসঙ্গে store করার জন্য JavaScript-এ Object ব্যবহার করা হয়।

# 4. Object কী

Instructor-এর definition:

> **“Objects are nothing but the key-value pair.”**

তিনি Object-কে আরও বলেছেন:

> **“A keyed collection.”**

অর্থাৎ Object হলো এমন একটি collection যেখানে:

- প্রতিটি data একটি unique `key` দিয়ে identify করা যায়।
- প্রতিটি `key`-এর বিপরীতে একটি `value` থাকে।
- পরে প্রয়োজন হলে সেই key ব্যবহার করে value retrieve করা যায়।

উদাহরণ:

```js
const user = {
  name: "Tapas",
  age: 40
};
```

এখানে:

| Property | Key | Value |
|---|---|---|
| প্রথম Property | `name` | `"Tapas"` |
| দ্বিতীয় Property | `age` | `40` |

## মনে রাখার নিয়ম

> Object = Related data-এর key-value collection.

---

# 5. Object Literal Syntax

Object তৈরির সবচেয়ে common ও straightforward উপায় হলো **Object Literal Syntax**।

## Empty Object

```js
const user = {};
```

শুধু `{}` লিখলে একটি Object তৈরি হয়।

কিন্তু এর মধ্যে কোনো information নেই বলে এটিকে বলা হয়:

> **Empty Object**

Object নিজেই একটি value। তাই number, string বা boolean-এর মতো এটিকেও variable-এ assign করা যায়।

```js
const emptyObject = {};
```

## Property-সহ Object

```js
let user = {
  name: "tapas",
  age: 40,
  "is adimin": true
};
```

এখানে:

- `{}` Object নির্দেশ করছে।
- `name`, `age`, `"is adimin"` হলো key।
- `"tapas"`, `40`, `true` হলো value।
- key এবং value-এর মধ্যে `:` ব্যবহার করা হয়।
- একাধিক property comma দিয়ে আলাদা করা হয়।

## Syntax

```js
const objectName = {
  key1: value1,
  key2: value2
};
```

## Common Mistake

ভুল:

```js
const user = {
  name = "Tapas",
  age = 40
};
```

সঠিক:

```js
const user = {
  name: "Tapas",
  age: 40
};
```

Object property-তে assignment operator `=` নয়, colon `:` ব্যবহার করতে হয়।

# 6. Object Property Access

Object-এর value ব্যবহার করতে property access করতে হয়।

JavaScript-এ প্রধানত দুইভাবে Object property access করা যায়:

1. Dot Notation
2. Bracket বা Subscript Notation

---

## 6.1 Dot Notation

```js
console.log(user.name);
console.log(user.age);
```

Output:

```text
tapas
40
```

Instructor-এর ভাষায়:

> Dot হলো এমন operator যার মাধ্যমে Object-এর একটি key-এর value query করা যায়।

## Syntax

```js
objectName.propertyName
```

উদাহরণ:

```js
const user = {
  name: "tapas",
  age: 40
};

console.log(user.name);
console.log(user.age);
```

---

## 6.2 Bracket Notation

```js
console.log(user["name"]);
console.log(user["age"]);
```

Bracket notation-এ property name সাধারণত string হিসেবে দিতে হয়।

```js
objectName["propertyName"]
```

Bracket notation বিশেষভাবে প্রয়োজন হয় যখন:

- key-তে space আছে
- key-তে special character আছে
- key dynamic variable থেকে আসছে

---

# 7. নতুন Property যোগ করা

Object তৈরি হওয়ার পরেও নতুন property যোগ করা যায়।

```js
user.isSeniorCitizen = false;
```

এখন `user` Object:

```js
{
  name: "tapas",
  age: 40,
  "is adimin": true,
  isSeniorCitizen: false
}
```

## Dot Notation দিয়ে

```js
user.isSeniorCitizen = false;
```

## Bracket Notation দিয়ে

```js
user["movie lover"] = true;
```

## মনে রাখার নিয়ম

> Existing Object-এর নতুন key-তে value assign করলেই নতুন property তৈরি হয়।

---

# 8. Special Character-যুক্ত Key

JavaScript Object-এর সাধারণ key quotation ছাড়া লেখা যায়:

```js
const user = {
  name: "Tapas",
  age: 40
};
```

JavaScript internally এই keyগুলোকে string হিসেবে treat করে।

কিন্তু key-তে space বা special character থাকলে quotation ব্যবহার করা বাধ্যতামূলক।

```js
const user = {
  "is admin": true,
  "movie lover": true
};
```

## Dot Notation কেন কাজ করে না

ভুল:

```js
console.log(user.is admin);
```

এটি syntax error দেবে।

কারণ JavaScript `user.is` পর্যন্ত একটি expression হিসেবে বুঝবে, এরপর `admin` আলাদা token হয়ে যাবে।

## Bracket Notation ব্যবহার করতে হবে

```js
console.log(user["is admin"]);
```

নতুন special-character key যোগ করতে:

```js
user["movie lover"] = true;
```

Modify করতে:

```js
user["movie lover"] = false;
```

Delete করতে:

```js
delete user["movie lover"];
```

## মনে রাখার নিয়ম

> Key-তে space থাকলে quotation + bracket notation ব্যবহার করো।

# 9. Property Modify ও Delete করা

## 9.1 Existing Property Modify

```js
user.age = 34;
```

আগে:

```js
age: 40
```

পরে:

```js
age: 34
```

Special-character key modify:

```js
user["movie lover"] = false;
```

---

## 9.2 Property Delete

JavaScript-এর `delete` keyword ব্যবহার করে Object property remove করা যায়।

```js
delete user.age;
```

Special-character key:

```js
delete user["movie lover"];
```

## গুরুত্বপূর্ণ পার্থক্য

```js
user.age = undefined;
```

এতে property delete হয় না। Property থাকে, শুধু value `undefined` হয়।

কিন্তু:

```js
delete user.age;
```

এতে property-টাই Object থেকে remove হয়।

# 10. Dynamic Key Access

Real-world application-এ property key সবসময় hardcoded থাকে না।

Key আসতে পারে:

- User input থেকে
- API response থেকে
- অন্য variable থেকে
- Program-এর runtime calculation থেকে

উদাহরণ:

```js
const someKey = "age";
```

এখন `someKey` variable-এর value হলো `"age"`।

## ভুল পদ্ধতি

```js
console.log(user.someKey);
```

এখানে JavaScript `"age"` property খুঁজবে না।

এটি `someKey` নামের literal property খুঁজবে।

## সঠিক পদ্ধতি

```js
console.log(user[someKey]);
```

Runtime-এ:

```js
user[someKey]
```

ব্যবহারিকভাবে হয়ে যায়:

```js
user["age"]
```

Output:

```text
34
```

## মনে রাখার নিয়ম

> Property name যদি variable থেকে আসে, dot notation নয়—bracket notation ব্যবহার করতে হবে।

# 11. Computed বা Dynamic Property Key দিয়ে Object তৈরি

শুধু dynamic key দিয়ে value access নয়, dynamic key দিয়ে Object property তৈরি করাও সম্ভব।

```js
let car = prompt("Which is your fav car?");

let favCars = {
  [car]: 5
};

console.log(favCars);
```

User যদি `BMW` লেখে:

```js
{
  BMW: 5
}
```

User যদি `Audi` লেখে:

```js
{
  Audi: 5
}
```

এখানে:

```js
[car]
```

কে বলা হয় computed property key।

## কেন bracket ব্যবহার করা হয়েছে

যদি লেখা হতো:

```js
const favCars = {
  car: 5
};
```

তাহলে key হতো literal `"car"`।

কিন্তু:

```js
const favCars = {
  [car]: 5
};
```

এখানে key হবে `car` variable-এর runtime value।

# 12. Object Creation Patterns

Instructor Object তৈরির একাধিক pattern দেখিয়েছেন।

| Pattern | `new` লাগে? | Custom Type পাওয়া যায়? | সাধারণ ব্যবহার |
|---|---:|---:|---|
| Object Literal | না | না | সবচেয়ে common |
| Constructor Function | হ্যাঁ | হ্যাঁ | Blueprint থেকে instance |
| `new Object()` | হ্যাঁ | না, শুধু generic Object | কম ব্যবহার |
| Factory Function | না | না | Function থেকে নতুন Object return |

---

## 12.1 Constructor Function

Constructor Function দেখতে regular function-এর মতো হলেও Object instance তৈরির জন্য ব্যবহার করা হয়।

```js
function Car(name, model) {
  this.name = name;
  this.model = model;
}
```

## Naming Convention

Instructor বলেছেন:

> Constructor Function-এর নাম convention অনুযায়ী capital letter দিয়ে শুরু করা উচিত।

সঠিক:

```js
function Car() {}
```

সাধারণ function:

```js
function createCar() {}
```

এটি language rule নয়, কিন্তু গুরুত্বপূর্ণ convention।

---

## Parameter ও Argument-এর সম্পর্ক

```js
function Car(name, model) {
  this.name = name;
  this.model = model;
}
```

এখানে:

- `name`, `model` হলো parameter বা placeholder।
- Object তৈরির সময় pass করা `"BMW"` ও `"X1"` হলো argument।

```js
const bmwCar = new Car("BMW", "X1");
```

---

## `new` Keyword

Constructor Function থেকে Object তৈরি করতে `new` ব্যবহার করতে হয়।

```js
const bmwCar = new Car("BMW", "X1");
const audiCar = new Car("Audi", "A8");
```

Output-এর ধারণা:

```js
Car {
  name: "BMW",
  model: "X1"
}
```

```js
Car {
  name: "Audi",
  model: "A8"
}
```

## Blueprint Analogy

Instructor Constructor Function-কে এমন একটি blueprint হিসেবে ব্যাখ্যা করেছেন, যেখান থেকে multiple Object instance তৈরি করা যায়।

- `Car` = blueprint
- `bmwCar` = একটি instance
- `audiCar` = আরেকটি instance

---

## `instanceof`

```js
console.log(bmwCar instanceof Car);
```

Output:

```text
true
```

এর অর্থ:

- `bmwCar` Objectটি `Car` constructor function থেকে তৈরি হয়েছে।

## Common Mistake

ভুল:

```js
const bmwCar = Car("BMW", "X1");
```

সঠিক:

```js
const bmwCar = new Car("BMW", "X1");
```

Constructor Function call করার সময় `new` না দিলে expected Object পাওয়া নাও যেতে পারে এবং `this`-এর behavior সমস্যাজনক হতে পারে।

---

## 12.2 Object Constructor

JavaScript-এর built-in `Object` constructor ব্যবহার করেও Object তৈরি করা যায়।

```js
const person = new Object();

person.name = "Alpha";
person.age = 76;

console.log(person);
```

Output:

```js
{
  name: "Alpha",
  age: 76
}
```

## Limitation

এখানে Object তৈরি হলেও custom type পাওয়া যায় না।

```js
person instanceof Object; // true
```

কিন্তু এটি `Car`, `Employee`, `Student` ইত্যাদি custom type নয়।

Instructor Object Literal-কে বেশি straightforward এবং common হিসেবে দেখিয়েছেন।

## 12.3 Factory Function

Instructor-এর explanation অনুযায়ী Factory Function এমন function যা input অনুযায়ী নতুন Object তৈরি করে return করতে পারে।

```js
function createUser(name, age) {
  return {
    name,
    age,
    greet() {
      console.log(this.name);
    }
  };
}
```

Object তৈরি:

```js
const user1 = createUser("tapas", 39);
const user2 = createUser("bob", 32);
```

এখানে `new` লাগে না।

## Constructor Function বনাম Factory Function

| বিষয় | Constructor Function | Factory Function |
|---|---|---|
| নাম | সাধারণত Capital letter | সাধারণ function naming |
| `new` লাগে | হ্যাঁ | না |
| Property তৈরি | `this.property` | Object return |
| Custom instance type | পাওয়া যায় | সাধারণত পাওয়া যায় না |
| Example | `new Car()` | `createUser()` |

## 12.4 Object Property Shorthand

পূর্ণ syntax:

```js
function createUser(name, age) {
  return {
    name: name,
    age: age
  };
}
```

যখন key name এবং variable/parameter name একই হয়, তখন shorthand ব্যবহার করা যায়।

```js
function createUser(name, age) {
  return {
    name,
    age
  };
}
```

Internally এটি এই অর্থই বহন করে:

```js
{
  name: name,
  age: age
}
```

## Rule

Shorthand ব্যবহার করা যাবে যখন:

```text
key name === variable name
```

# 15. Property Exist করে কি না পরীক্ষা

## 15.1 Direct Value Check-এর সমস্যা

```js
console.log(profile.salary);
```

```js
if (!profile.salary) {
```

তবুও:

truthy হয়ে message দেখাবে যে property নেই।

এটি ভুল conclusion।

# 16. `for...in` Loop

Object-এর সব enumerable property key iterate করতে `for...in` loop ব্যবহার করা যায়।

```js
for (let key in profile) {
  console.log(key);
}
```

এটি keyগুলো print করবে।

Value access করতে:

```js
for (let key in profile) {
  console.log(key);
  console.log(profile[key]);
}
```

এখানে `key` dynamic variable। তাই:

```js
profile[key]
```

ব্যবহার করা হয়েছে।

`profile.key` ব্যবহার করলে literal `key` নামের property খোঁজা হবে।

## Syntax

```js
for (let key in object) {
  // key ব্যবহার
}
```

## Common Mistake

ভুল:

```js
for (let key in profile) {
  console.log(profile.key);
}
```

সঠিক:

```js
for (let key in profile) {
  console.log(profile[key]);
}
```

---

## `for...in` বনাম `for...of`

| Loop | সাধারণত কী iterate করে |
|---|---|
| `for...in` | Object-এর keys |
| `for...of` | Iterable-এর values, যেমন Array |

Object destructuring section-এ Array of Objects-এর জন্য `for...of` ব্যবহার করা হয়েছে।

# 18. Object Reference ও Object Comparison

Object একটি **non-primitive data type**।

Instructor আগের Execution Context ও memory discussion-এর সঙ্গে এই topic যুক্ত করেছেন।

## মূল ধারণা

> Object value সরাসরি variable-এর মধ্যে primitive value-এর মতো copy না হয়ে reference-এর মাধ্যমে handle হয়।

একটি Object memory-এর একটি location-এ stored থাকে। Variable সেই Object-এর reference ধরে।

উদাহরণ:

```js
let fruit = { name: "mango" };
const oneMoreFruit = { name: "mango" };
```

দুই Object-এর content একই:

```js
{ name: "mango" }
```

তবুও:

```js
console.log(fruit == oneMoreFruit);  // false
console.log(fruit === oneMoreFruit); // false
```

কারণ:

- `fruit` একটি memory location-এর Object refer করছে।
- `oneMoreFruit` অন্য memory location-এর Object refer করছে।
- Value দেখতে একই হলেও reference আলাদা।

Instructor উদাহরণ হিসেবে আলাদা memory address-এর ধারণা দিয়েছেন:

```text
fruit        -> XA01
oneMoreFruit -> YB02
```

---

## একই Reference Assign করা

```js
fruit = oneMoreFruit;
```

এখন দুই variable একই Object refer করছে।

```js
console.log(fruit == oneMoreFruit);  // true
console.log(fruit === oneMoreFruit); // true
```

## `let` কেন প্রয়োজন

প্রথম variable যদি `const` হয়:

```js
const fruit = { name: "mango" };
fruit = oneMoreFruit;
```

তাহলে reassignment error হবে।

তাই reference reassign করার demonstration-এ `let` ব্যবহার করা হয়েছে।

## মনে রাখার নিয়ম

> Object equality content দিয়ে নয়, reference দিয়ে compare হয়।

# 19. Object Static Methods

Instructor এমন কয়েকটি Object static method আলোচনা করেছেন যেগুলো real-world programming-এ নিয়মিত ব্যবহৃত হয়।

Static method Object instance-এর ওপর নয়, built-in `Object` constructor-এর ওপর call করা হয়।

উদাহরণ:

```js
Object.keys(...)
Object.assign(...)
Object.entries(...)
```

---

## 19.1 `Object.assign()`

Instructor-এর definition:

> `Object.assign()` source Object-এর own properties target Object-এ copy করে।

## Syntax

```js
Object.assign(target, source);
```

উদাহরণ:

```js
const target = { p: 1, a: 2 };
const source = { a: 3, b: 5 };

const returnedObj = Object.assign(target, source);

console.log(returnedObj);
```

Output:

```js
{
  p: 1,
  a: 3,
  b: 5
}
```

## Override Rule

যদি target ও source-এ same key থাকে, source-এর value target-এর value override করে।

এখানে:

```js
target.a = 2
source.a = 3
```

Final:

```js
a: 3
```

---

## Target Mutate হয়

```js
Object.assign(target, source);
```

এটি target Object modify করে।

```js
console.log(target === returnedObj); // true
```

এই point lecture code-এ সরাসরি compare না করা হলেও `Object.assign()` বোঝার জন্য গুরুত্বপূর্ণ।

---

## Clone-এর জন্য Empty Target

```js
const obj = {
  name: "tapaScript"
};

const obj2 = Object.assign({}, obj);
```

এখানে:

- `{}` নতুন target Object
- `obj` source Object

```js
console.log(obj2);
```

Output:

```js
{
  name: "tapaScript"
}
```

কিন্তু:

```js
console.log(obj === obj2);
```

Output:

```text
false
```

কারণ values একই হলেও reference আলাদা।

---

## 19.2 Shallow Copy বনাম Deep Copy

### Shallow Copy

Instructor-এর explanation:

> `Object.assign()` shallow copy করে।

এর অর্থ:

- Top-level primitive property নতুন Object-এ copy হয়।
- Nested Object-এর নতুন deep copy তৈরি হয় না।
- Nested Object-এর reference copy হয়।

উদাহরণ:

```js
const obj3 = {
  a: 1,
  b: {
    c: 2
  }
};

const obj4 = Object.assign({}, obj3);
```

এখন:

```js
obj4.a = 100;
```

এতে শুধু `obj4.a` বদলাবে।

```js
console.log(obj4.a); // 100
console.log(obj3.a); // 1
```

কিন্তু:

```js
obj4.b.c = 3;
```

তখন:

```js
console.log(obj4.b.c); // 3
console.log(obj3.b.c); // 3
```

কারণ `obj3.b` ও `obj4.b` একই nested Object reference করছে।

### Visual Model

```text
obj3 -----> {
               a: 1,
               b: -----> { c: 2 }
             }

obj4 -----> {
               a: 1,
               b: -------^
             }
```

Top-level Object আলাদা, কিন্তু nested `b` একই reference।

---

### Deep Copy

Deep Copy-তে:

- Top-level Object নতুন হয়।
- Nested Object-ও নতুন reference পায়।
- Target-এর nested value change করলে source পরিবর্তিত হয় না।

---

## 19.3 `structuredClone()`

Deep cloning-এর জন্য Instructor `structuredClone()` ব্যবহার করেছেন।

```js
const obj5 = structuredClone(obj3);
```

এরপর:

```js
obj5.a = 300;
obj5.b.c = 30;
```

Result:

```js
console.log(obj5.a);   // 300
console.log(obj3.a);   // 1

console.log(obj5.b.c); // 30
console.log(obj3.b.c); // 2
```

## কেন কাজ করছে

`structuredClone()` nested data-সহ separate copy তৈরি করে।

```text
obj3.b !== obj5.b
```

## কখন ব্যবহার করবে

- Nested Object clone করতে
- Nested Array clone করতে
- Source Object mutate হওয়া আটকাতে

## গুরুত্বপূর্ণ Note

সব ধরনের value `structuredClone()` দিয়ে clone করা যায় না। Function clone করা যায় না। তবে Instructor-এর current example-এর plain nested data-এর ক্ষেত্রে এটি সঠিক tool।

## 19.4 `Object.entries()`

Instructor-এর explanation:

> `Object.entries()` Object-কে Array representation-এ convert করতে ব্যবহার করা যায়।

উদাহরণ:

```js
const myObj = {

const myArr = Object.entries(myObj);

```

Output:

```js
[
  ["a", "tapas"],

```

প্রতিটি property একটি two-element Array হয়:

```text
[key, value]
```

## Practical Use

Instructor API/UI example দিয়েছেন:

- API Object return করতে পারে।
- UI loop করার জন্য Array চাইতে পারে।
- `Object.entries()` দিয়ে Object-কে loop-friendly Array-তে convert করা যায়।

## Loop Example

```js
for (const [key, value] of Object.entries(myObj)) {
  console.log(key, value);
}
```

```js
const entries = new Map([
```

```js
const data = [
```

## 19.6 `Object.freeze()`

Instructor-এর definition-এর অর্থ:

> Immutable মানে এমন কিছু যা change করা যায় না। `Object.freeze()` Object-কে এমনভাবে freeze করে যে existing property modify, নতুন property add, বা property delete করা যায় না।

উদাহরণ:

```js
const emp = {
  sal: 100
};

Object.freeze(emp);
```

এখন:

```js
emp.sal = 200;
emp.name = "Alex";
delete emp.sal;
```

সাধারণ non-strict mode-এ এগুলো silently fail করতে পারে।

```js
console.log(emp);
```

Output:

```js
{
  sal: 100
}
```

## Frozen কি না Check

```js
console.log(Object.isFrozen(emp));
```

Output:

```text
true
```

## Freeze Behavior Table

| Operation | Allowed? |
|---|---:|
| Existing property modify | না |
| New property add | না |
| Property delete | না |

## Important Depth Note

`Object.freeze()` সাধারণত shallow।

```js
const obj = {
  nested: {
    value: 1
  }
};

Object.freeze(obj);

obj.nested.value = 2;
```

Nested Object আলাদাভাবে freeze না করলে inner value change হতে পারে। এটি lecture-এর মূল example-এর বাইরে একটি গুরুত্বপূর্ণ implementation note।

## 19.7 `Object.seal()`

`Object.seal()` Object-এর structure lock করে।

Instructor-এর explanation অনুযায়ী:

- নতুন property add করা যায় না।
- existing property delete করা যায় না।
- existing property-এর value modify করা যায়।

উদাহরণ:

```js
const dept = {
  name: "finance"
};

Object.seal(dept);
```

### Add করার চেষ্টা

```js
dept.address = "Bangalore";
```

Add হবে না।

### Delete করার চেষ্টা

```js
delete dept.name;
```

Delete হবে না।

### Existing Value Modify

```js
dept.name = "HR";
```

এটি কাজ করবে।

```js
console.log(dept);
```

Output:

```js
{
  name: "HR"
}
```

---

## Freeze বনাম Seal

| Operation | `Object.freeze()` | `Object.seal()` |
|---|---:|---:|
| Existing property modify | না | হ্যাঁ |
| New property add | না | না |
| Property delete | না | না |

## মনে রাখার নিয়ম

> Freeze = Value ও structure দুটোই lock  
> Seal = Structure lock, existing value change করা যায়

## 19.8 `Object.hasOwn()`

Instructor-এর explanation:

> `Object.hasOwn()` কোনো Object-এর নিজের property আছে কি না, তার ভিত্তিতে `true` বা `false` return করে।

উদাহরণ:

```js
const dept = {
  name: "finance"
};

console.log(Object.hasOwn(dept, "name"));
```

Output:

```text
true
```

```js
console.log(Object.hasOwn(dept, "address"));
```

Output:

```text
false
```

## Syntax

```js
Object.hasOwn(object, propertyName)
```

## `in` বনাম `Object.hasOwn()`

| Method | কী check করে |
|---|---|
| `"key" in object` | Own + inherited property |
| `Object.hasOwn(object, "key")` | শুধু Object-এর own property |

Prototype শেখার পর এই পার্থক্য আরও গুরুত্বপূর্ণ হবে।

# 20. Object Destructuring

Instructor-এর conceptual explanation:

> Destructure মানে কোনো structure ভেঙে প্রয়োজনীয় অংশ আলাদা করে নেওয়া।

Object-এর ক্ষেত্রে:

- Object-এর property থেকে value বের করা
- সেই value দিয়ে variable তৈরি করা

## Without Destructuring

```js
const name = student.name;
const city = student.address.city;
```

এটি কাজ করে, কিন্তু বেশি code লিখতে হয়।

## With Destructuring

```js
const { name, age } = student;
```

JavaScript তৈরি করবে:

```text
name = student.name
age = student.age
```

## Example Object

```js
const student = {
  name: "John Williamson",
  age: 9,
  std: 3,
  subjects: ["Maths", "English", "EVS"],

  parents: {
    father: "Brown Williamson",
    mother: "Sophia",
    email: "john-parents@abcde.com"
  },

  address: {
    street: "65/2, brooklyn road",
    city: "Carterton",
    country: "New Zealand",
    zip: 5791
  }
};
```

এই Object-এ আছে:

- Primitive value
- Array
- Nested Object

এই কারণেই Instructor destructuring-এর বিভিন্ন use case দেখাতে এটিকে ব্যবহার করেছেন।

---

# 21. Destructuring-এর Practical Use Cases

---

## 21.1 Basic Property Extraction

```js
const { name, age } = student;

console.log(name);
console.log(age);
```

Output:

```text
John Williamson
9
```

## Important Rule

Destructuring-এর variable name সাধারণত Object property key-এর সঙ্গে match করতে হয়।

```js
const { name } = student;
```

`student.name` পাওয়া যাবে।

---

## 21.2 Default Value

Object-এ property না থাকলেও destructuring-এর সময় default value দেওয়া যায়।

```js
const { name, age, meal = "bread" } = student;
```

`student` Object-এ `meal` নেই।

তাই:

```js
meal === "bread"
```

Output:

```text
John Williamson 9 bread
```

## যদি Property থাকে

```js
const student = {
  meal: "rice"
};

const { meal = "bread" } = student;

console.log(meal);
```

Output:

```text
rice
```

Default value শুধু তখনই ব্যবহৃত হয় যখন property value `undefined`।

## Equivalent Longer Logic

```js
const meal =
  student.meal !== undefined
    ? student.meal
    : "bread";
```

Destructuring syntax একই কাজ অনেক সংক্ষিপ্তভাবে করে।

---

## 21.3 Dynamic Default Value

একটি destructured variable ব্যবহার করে আরেকটির default value calculate করা যায়।

```js
const {
  subjects,
  numberOfSubjects = subjects.length
} = student;
```

এখানে:

```js
subjects
```

হবে:

```js
["Maths", "English", "EVS"]
```

তারপর:

```js
subjects.length
```

হবে:

```text
3
```

Output:

```js
console.log(numberOfSubjects); // 3
```

## Order গুরুত্বপূর্ণ

```js
const {
  subjects,
  numberOfSubjects = subjects.length
} = student;
```

এখানে `subjects` আগে destructure হয়েছে।

---

## 21.4 Alias বা Rename

একটি property destructure করে অন্য variable name ব্যবহার করা যায়।

```js
const { std: standard } = student;
```

এখানে:

- `std` = source property key
- `standard` = নতুন variable name বা alias

```js
console.log(standard); // 3
```

## গুরুত্বপূর্ণ

এক্ষেত্রে `std` নামে variable তৈরি হয় না।

```js
console.log(std);
```

ReferenceError হবে।

## Syntax

```js
const { propertyName: aliasName } = object;
```

---

## Alias কেন দরকার

Instructor API conflict example দিয়েছেন।

বিভিন্ন source থেকে একই নামের property আসতে পারে:

- GraphQL response
- REST response
- Webhook response

তিনটিতেই `loading` property থাকলে একই scope-এ conflict হতে পারে।

```js
const { loading: graphQLLoading } = graphQLResponse;
const { loading: restLoading } = restResponse;
const { loading: webhookLoading } = webhookResponse;
```

---

## 21.5 Nested Object Destructuring

Nested Object থেকে সরাসরি value বের করা যায়।

```js
const {
  address: {
    zip
  }
} = student;
```

এখন:

```js
console.log(zip); // 5791
```

## Syntax বুঝি

```js
address: { zip }
```

অর্থ:

1. `student.address`-এ যাও
2. তার ভিতর থেকে `zip` বের করো

## Important Mistake

এই syntax-এ সাধারণত `address` নামে variable তৈরি হয় না।

```js
const { address: { zip } } = student;
```

এখানে `zip` আছে, কিন্তু `address` variable আলাদাভাবে নেই।

দুটোই দরকার হলে:

```js
const { address, address: { zip } } = student;
```

---

## 21.6 Function Parameter-এ Destructuring

Function-এ পুরো Object pass করলেও parameter level-এ শুধু প্রয়োজনীয় অংশ destructure করা যায়।

### Without Parameter Destructuring

```js
function sendEmail(student) {
  console.log(
    `Sent an email to ${student.parents.email}`
  );
}

sendEmail(student);
```

### With Parameter Destructuring

```js
function sendEmail({
  parents: {
    email
  }
}) {
  console.log(`Sent an email to ${email}`);
}

sendEmail(student);
```

Output:

```text
Sent an email to john-parents@abcde.com
```

## Benefit

- Function body cleaner হয়।
- Repeated long property chain লিখতে হয় না।
- Function কোন data ব্যবহার করছে তা parameter দেখেই বোঝা যায়।

---

## 21.7 Function Return Value Destructuring

Function যদি Object return করে, call করার সময়ই destructure করা যায়।

```js
const getStudent = () => {
  return {
    name: "John Williamson",
    age: 9,
    std: 3,
    subjects: ["Maths", "English", "EVS"],

    parents: {
      father: "Brown Williamson",
      mother: "Sophia",
      email: "john-parents@abcde.com"
    },

    address: {
      street: "65/2, brooklyn road",
      city: "Carterton",
      country: "New Zealand",
      zip: 5791
    }
  };
};
```

Direct destructuring:

```js
const {
  name: anotherName,
  subjects: anotherSubs
} = getStudent();
```

Output:

```js
console.log(anotherName);
// John Williamson

console.log(anotherSubs);
// ["Maths", "English", "EVS"]
```

এখানে alias প্রয়োজন হয়েছে, কারণ আগেই `name` বা `subjects` variable declare করা থাকতে পারে।

---

## 21.8 Loop-এর মধ্যে Destructuring

Array of Objects:

```js
const students = [
  {
    name: "William",
    grade: "A"
  },
  {
    name: "Tom",
    grade: "A+"
  },
  {
    name: "Bob",
    grade: "B"
  }
];
```

`for...of` loop-এ প্রতিটি Object destructure:

```js
for (let { name, grade } of students) {
  console.log(name, grade);
}
```

Output:

```text
William A
Tom A+
Bob B
```

## Flow

প্রতিটি iteration-এ:

1. Array থেকে একটি student Object পাওয়া যায়।
2. Object থেকে `name` ও `grade` destructure হয়।
3. সেই valueগুলো print হয়।

# 22. Optional Chaining

Instructor বলেছেন Optional Chaining ES2020-এ introduced হয়েছে।

এটি nested property access করার সময় `null` বা `undefined` থাকলে runtime error এড়াতে সাহায্য করে।

## Problem Example

```js
const employee = {
  salary: {
    bonus: 300
  }
};
```

`department` property নেই।

```js
console.log(employee.department);
```

Output:

```text
undefined
```

কিন্তু:

```js
console.log(employee.department.name);
```

Error:

```text
Cannot read properties of undefined
```

কারণ:

```js
employee.department
```

আগেই `undefined`।

`undefined.name` access করা যায় না।

---

## Traditional Guard Check

```js
const name =
  employee.department &&
  employee.department.name;
```

এটি কাজ করতে পারে, কিন্তু chain বড় হলে code দীর্ঘ হয়।

---

## Optional Chaining Syntax

```js
const name = employee.department?.name;
```

এখানে:

```js
?.
```

হলো Optional Chaining Operator।

Output:

```text
undefined
```

কিন্তু application runtime error দিয়ে terminate করে না।

---

## কীভাবে কাজ করে

```js
employee.department?.name
```

Logic:

1. `employee.department` nullish কি না check করো।
2. যদি `null` বা `undefined` হয়, পুরো expression `undefined` return করো।
3. না হলে `.name` access করো।

---

## Multiple Level

```js
const city =
  user?.profile?.address?.city;
```

যেকোনো intermediate level `null` বা `undefined` হলে error-এর বদলে `undefined` return হবে।

---

## কখন Optional Chaining ব্যবহার করবে

- API data incomplete হতে পারে
- Nested property optional
- User data-এর কিছু অংশ missing
- UI crash না করিয়ে gracefully fallback দরকার

---

## কখন ব্যবহার না করাও যুক্তিযুক্ত

Instructor বলেছেন কখনো developer ইচ্ছাকৃতভাবে error দেখতে চাইতে পারে।

যদি কোনো property অবশ্যই থাকার কথা এবং না থাকলে bug report হওয়া দরকার, তখন optional chaining ব্যবহার না করাই ভালো হতে পারে।

উদাহরণ:

```js
const name = employee.department.name;
```

এতে missing data দ্রুত ধরা পড়বে।

## মনে রাখার নিয়ম

> Optional Chaining missing data gracefully handle করে; কিন্তু সব জায়গায় blindly ব্যবহার করা উচিত নয়।

# 23. Common Mistakes

## Mistake 1: Object property-তে `=` ব্যবহার

ভুল:

```js
const user = {
  name = "John"
};
```

সঠিক:

```js
const user = {
  name: "John"
};
```

---

## Mistake 2: Space-যুক্ত key dot notation দিয়ে access

ভুল:

```js
user.movie lover
```

সঠিক:

```js
user["movie lover"]
```

---

## Mistake 3: Dynamic key dot notation দিয়ে access

ভুল:

```js
const key = "age";
console.log(user.key);
```

সঠিক:

```js
console.log(user[key]);
```

---

## Mistake 4: Computed key-তে bracket না দেওয়া

ভুল:

```js
const car = "BMW";

const favCars = {
  car: 5
};
```

এখানে key `"car"` হবে।

সঠিক:

```js
const favCars = {
  [car]: 5
};
```

এখানে key `"BMW"` হবে।

---

## Mistake 5: Constructor Function-এ `new` ভুলে যাওয়া

ভুল:

```js
const car = Car("BMW", "X1");
```

সঠিক:

```js
const car = new Car("BMW", "X1");
```

---

## Mistake 6: Object content equality আশা করা

```js
{ name: "mango" } === { name: "mango" }
```

Output:

```text
false
```

কারণ references আলাদা।

---

## Mistake 7: `Object.assign()`-কে deep clone ভাবা

```js
const copy = Object.assign({}, original);
```

Nested Object থাকলে এটি shallow copy।

Deep copy দরকার হলে plain structured data-এর জন্য:

```js
const copy = structuredClone(original);
```

---

## Mistake 8: Property existence falsy check দিয়ে করা

ভুল:

```js
if (!profile.salary) {
  // property নেই ধরে নেওয়া
}
```

কিন্তু salary `0` হলেও condition true হবে।

সঠিক:

```js
if ("salary" in profile) {
}
```

অথবা own property-এর জন্য:

```js
if (Object.hasOwn(profile, "salary")) {
}
```

---

## Mistake 9: Destructuring alias-এর পর original নাম ব্যবহার

```js
const { std: standard } = student;

console.log(std);
```

Error হবে।

সঠিক:

```js
console.log(standard);
```

---

## Mistake 10: Nested destructuring-এ parent variable আছে ধরে নেওয়া

```js
const { address: { zip } } = student;

console.log(address);
```

`address` variable তৈরি হয়নি।

---

## Mistake 11: `for...in`-এ value access করতে dot notation

ভুল:

```js
for (let key in profile) {
  console.log(profile.key);
}
```

সঠিক:

```js
for (let key in profile) {
  console.log(profile[key]);
}
```

---

## Mistake 12: Optional Chaining-এর পর value নিশ্চিত ধরে নেওয়া

```js
const city = user?.address?.city;
console.log(city.toUpperCase());
```

`city` যদি `undefined` হয়, error হবে।

সতর্কভাবে:

```js
console.log(city?.toUpperCase());
```

অথবা fallback:

```js
console.log(city ?? "Unknown City");
```

# 24. Interview Questions

## 1. JavaScript Object কী?

Object হলো key-value pair-এর keyed collection, যেখানে unique key ব্যবহার করে value store ও retrieve করা যায়।

## 2. Object Literal Syntax কী?

Curly braces `{}` ব্যবহার করে সরাসরি Object তৈরির syntax।

```js
user.name
```

Bracket notation special-character key বা dynamic key-এর জন্য।

```js
user["movie lover"]
user[key]
```

---

## 4. Computed Property কী?

Variable-এর runtime value ব্যবহার করে Object key তৈরি করা।

```js
const key = "age";

const obj = {
  [key]: 9
};
```

---

## 5. Constructor Function কী?

একটি function যা `new` keyword-এর মাধ্যমে Object instance তৈরির blueprint হিসেবে ব্যবহৃত হয়।

---

## 6. Factory Function কী?

একটি regular function যা একটি নতুন Object return করে।

```js
function createUser(name) {

}
```

---

## 7. Constructor ও Factory Function-এর মূল পার্থক্য কী?

Constructor Function call করতে `new` লাগে এবং custom instance type পাওয়া যায়। Factory Function regular function-এর মতো call হয় এবং Object return করে।

## 8. Object Method কী?

Object-এর property হিসেবে থাকা function-কে method বলা হয়।

## 9. Nested Object কী?

## 10. `in` Operator কী করে?

## 11. `Object.hasOwn()` কী করে?

শুধু Object-এর own property আছে কি না check করে।

## 12. দুইটি একই content-এর Object কেন `===`-এ false?

কারণ Object reference দিয়ে compare হয়, content দিয়ে নয়।

---

## 13. `Object.assign()` কী করে?

Source Object-এর own enumerable properties target Object-এ copy করে।

---

## 14. Shallow Copy কী?

Top-level property copy হলেও nested Object-এর reference shared থাকে।

## 15. Deep Copy কী?

Nested Object-সহ সম্পূর্ণ independent copy।
